# Codebase Structure

**Analysis Date:** 2026-04-16

## Directory Layout

```
zotero-cli/
+-- src/
|   +-- main.rs        # CLI entry point, arg parsing, command dispatch, output format selection
|   +-- client.rs      # ZoteroClient: all HTTP calls to Zotero local connector API
|   +-- config.rs      # Config struct: TOML load, env var overrides, defaults
|   +-- types.rs       # Domain types: ZoteroItem, ItemData, Creator, Tag, ZoteroCollection, CompactItem
|   +-- output.rs      # Table renderers for terminal output (tabled + colored)
|   +-- merge.rs       # Pure merge logic: reconcile_items, build_dry_run_report
+-- bench/
|   +-- bench.py       # Python benchmark script (latency/token measurements)
|   +-- plots/         # Output plots from bench.py
+-- .planning/
|   +-- codebase/      # Codebase map documents (this directory)
+-- Cargo.toml         # Package manifest and dependencies
+-- Cargo.lock         # Locked dependency versions
```

## Directory Purposes

**`src/`:**
- Purpose: All Rust source code; single binary target
- Contains: 6 modules, all co-located at flat depth (no subdirectories)
- Key files: see module descriptions below

**`bench/`:**
- Purpose: Performance benchmarking (latency and token cost)
- Contains: `bench.py` script, `.venv/` (Python venv), `plots/` (output images)
- Generated: plots are generated; `.venv/` is gitignored
- Committed: `bench.py`, `plots/*.png`

**`.planning/codebase/`:**
- Purpose: Codebase map documents for GSD planning/execution workflow
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other focus-area docs
- Generated: Yes, by GSD map commands
- Committed: Yes

## Key File Locations

**Entry Points:**
- `src/main.rs`: Binary entry point; `fn main` and `fn run`; clap `Cli`/`Commands` definitions

**Configuration:**
- `Cargo.toml`: Package metadata, all dependency versions, release profile (strip, LTO, size-optimize)
- `src/config.rs`: Runtime config struct; default API base `http://localhost:23119/api`; config file path `~/.config/zotero-cli/config.toml`

**Core Logic:**
- `src/client.rs`: All HTTP operations; URL construction with `lib_path()` helper; `ZoteroClient::new` takes `&Config`
- `src/merge.rs`: `reconcile_items` (merge strategy) + `build_dry_run_report` (preview); no I/O
- `src/types.rs`: All serde-annotated structs; `#[serde(flatten)] extra` in `ItemData` for lossless round-trip

**Rendering:**
- `src/output.rs`: One public function per entity type (`items_table`, `item_detail`, `annotations_table`, `notes_table`, `collections_table`, `tags_table`); all return `String`

**Testing:**
- `src/config.rs` `#[cfg(test)]` block: config parsing, roundtrip, defaults
- `src/types.rs` `#[cfg(test)]` block: `Creator::display_name`, `CompactItem::from_item`, serde roundtrip
- `src/output.rs` `#[cfg(test)]` block: `truncate`, `wrap`, `strip_html`, all table renderers
- `src/merge.rs` `#[cfg(test)]` block: merge strategy, tag/collection union, structural field protection, dry-run report

## Naming Conventions

**Files:**
- snake_case module names matching the primary abstraction they contain: `client.rs`, `config.rs`, `merge.rs`, `output.rs`, `types.rs`

**Directories:**
- lowercase, no hyphens for Rust source; kebab-case for non-Rust dirs (`bench/`)

## Where to Add New Code

**New subcommand:**
- Add variant to `Commands` enum in `src/main.rs`
- Add handler arm in the `match cli.command` block in `src/main.rs`
- If it calls new API endpoints: add method(s) to `ZoteroClient` in `src/client.rs`
- If it needs a new output format: add a function to `src/output.rs`
- If it needs new types: add structs to `src/types.rs`

**New API endpoint:**
- Implementation: `src/client.rs` as a method on `ZoteroClient`
- Use `get_json` / `post_json` / `patch_json` private helpers
- Return typed structs from `src/types.rs` or `serde_json::Value` for unstructured responses

**New type/struct:**
- Implementation: `src/types.rs`
- Derive `Deserialize`, `Serialize`, `Debug`, `Clone` by default
- Use `#[serde(rename_all = "camelCase")]` for types mirroring Zotero API shapes

**New output renderer:**
- Implementation: `src/output.rs` as a free function returning `String`
- Use `tabled::builder::Builder` + `Style::modern()` for tables; `colored` for emphasis

**New config field:**
- Add to `Config` struct in `src/config.rs` with `#[serde(default = "...")]`
- Add `Default` impl value and env var override in `Config::load()`

**Tests:**
- Co-located in `#[cfg(test)]` module at bottom of the same file as the code under test
- No separate `tests/` directory; no integration test harness currently

## Special Directories

**`bench/`:**
- Purpose: Python-based latency and token benchmarks; not part of the Rust build
- Generated: `plots/` images are generated by `bench.py`
- Committed: Script and plots yes; `.venv/` no

**`target/`:**
- Purpose: Rust build artifacts (debug, release, test binaries)
- Generated: Yes
- Committed: No (gitignored)

## Module Dependency Graph

```
main.rs
  +-- config.rs      (Config)
  +-- client.rs      (ZoteroClient)
  |     +-- config.rs  (Config)
  |     +-- types.rs   (ZoteroItem, ZoteroCollection)
  +-- types.rs       (CompactItem, ZoteroItem)
  +-- output.rs      (table renderers)
  |     +-- types.rs   (ZoteroItem, ZoteroCollection)
  +-- merge.rs       (reconcile_items, build_dry_run_report)
        +-- types.rs   (ZoteroItem)
```

`types.rs` is a leaf with no intra-crate imports. `config.rs` is a leaf with no intra-crate imports. No circular dependencies.
